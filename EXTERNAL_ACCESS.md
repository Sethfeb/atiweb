# 외부 접속 설정 가이드

## 개발 서버를 외부에서 접속 가능하게 설정

### 1. 서버 시작 방법

터미널에서 다음 명령어를 실행하세요:

```bash
cd /Users/seth/ati-corporate-website
npm run dev
```

서버는 자동으로 `0.0.0.0`에 바인딩되어 외부 접속이 가능합니다.

### 2. IP 주소 확인

터미널에서 다음 명령어로 로컬 IP 주소를 확인하세요:

```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

또는:

```bash
ipconfig getifaddr en0  # Wi-Fi의 경우
ipconfig getifaddr en1  # 이더넷의 경우
```

### 3. 접속 방법

서버가 시작되면 다음과 같이 접속할 수 있습니다:

- **로컬 접속**: http://localhost:3000/en/
- **같은 네트워크 내 다른 기기**: http://[로컬IP]:3000/en/
  - 예: http://192.168.0.100:3000/en/

### 4. 방화벽 설정

macOS 방화벽이 활성화되어 있다면, 터미널 앱에 대한 네트워크 접속을 허용해야 할 수 있습니다:

1. 시스템 설정 > 네트워크 > 방화벽
2. 방화벽 옵션 클릭
3. "자동으로 서명된 소프트웨어가 수신 연결을 허용하도록 허용" 체크

### 5. 포트 포워딩 (인터넷에서 접속하려는 경우)

인터넷에서 접속하려면 라우터에서 포트 포워딩을 설정해야 합니다:

1. 라우터 관리 페이지 접속 (보통 192.168.0.1 또는 192.168.1.1)
2. 포트 포워딩 설정
3. 외부 포트: 3000 (또는 원하는 포트)
4. 내부 IP: [Mac의 로컬 IP]
5. 내부 포트: 3000

그 후 공인 IP 주소로 접속:
- http://[공인IP]:3000/en/

### 6. 동적 DNS 사용 (선택사항)

공인 IP가 자주 변경되는 경우, 동적 DNS 서비스를 사용할 수 있습니다:
- No-IP (https://www.noip.com/)
- Duck DNS (https://www.duckdns.org/)

### 7. 보안 주의사항

⚠️ **중요**: 개발 서버는 프로덕션 환경용이 아닙니다. 외부에 노출할 때는 다음을 주의하세요:

- 개발 서버는 보안 기능이 제한적입니다
- 프로덕션 배포 시에는 Vercel, Netlify 등의 호스팅 서비스를 사용하세요
- 민감한 정보가 포함된 환경 변수는 절대 커밋하지 마세요

### 8. 프로덕션 배포 (권장)

실제 서비스로 운영하려면:

1. **Vercel 배포** (권장):
   ```bash
   npm install -g vercel
   vercel
   ```

2. **Netlify 배포**:
   - GitHub 저장소 연결
   - 빌드 명령: `npm run build`
   - 출력 디렉토리: `.next`

3. **자체 서버 배포**:
   ```bash
   npm run build
   npm start
   ```

### 현재 설정

- 서버 바인딩: `0.0.0.0` (모든 네트워크 인터페이스)
- 포트: `3000`
- 프로토콜: HTTP
